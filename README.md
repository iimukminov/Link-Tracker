# Link Tracker (Bot & Scrapper)

Система мониторинга обновлений в **GitHub** (Issues, PR) и **StackOverflow** (ответы, комментарии) с автоматическим уведомлением пользователя через Telegram-бота.

## 🚀 Быстрый запуск

Проект полностью подготовлен к работе в Docker-контейнерах. Вся инфраструктура, включая базу данных и микросервисы, настраивается через единый файл окружения.

### 1. Настройка окружения
Создайте файл `.env` в корневом каталоге проекта, скопировав содержимое шаблона `env_dist`:
```bash
cp env_dist .env
```
В файле `.env` необходимо заполнить следующие параметры:
* **Telegram**: `TELEGRAM_BOT_TOKEN` — токен вашего бота от [@BotFather](https://t.me/BotFather).
* **GitHub**: `GITHUB_TOKEN` — Personal Access Token для работы с API.
* **StackOverflow**: `STACKOVERFLOW_KEY` и `STACKOVERFLOW_ACCESS_KEY` — ключи доступа к API.
* **Database**: По умолчанию установлены стандартные значения (`postgres/postgres`), которых достаточно для локального запуска.

### 2. Запуск системы
Запустите все компоненты одной командой:
```bash
docker compose up -d
```
Docker Compose автоматически выполнит:
1. Поднятие контейнера **PostgreSQL**.
2. Применение миграций **Liquibase** для создания схем и индексов.
3. Сборку и запуск микросервисов **Bot** и **Scrapper**.

## 🛠 Особенности реализации

* **Параллельная обработка**: Scrapper использует многопоточный пул для одновременной проверки батчей ссылок (количество потоков настраивается).
* **Изоляция сбоев**: Ошибка при проверке одной ссылки не блокирует работу планировщика. Бот отправляет пользователю отчет, если ресурс временно недоступен.
* **Гибкость данных**: Поддержка двух режимов доступа к БД через параметр `APP_DATABASE_ACCESS_TYPE`:
    * `SQL` — прямое взаимодействие через JDBC.
    * `ORM` — использование JPA (Hibernate).
* **Сетевое взаимодействие**: Внутри Docker-сети сервисы общаются по именам (`http://scrapper:8081` и `http://bot:8080`), что исключает конфликты портов на хосте.
